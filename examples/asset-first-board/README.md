## What This Example Demonstrates / 这个示例说明什么

- First screen is a visual contact sheet, not a long summary.
- Assets appear before page planning prose.
- Every asset has role, crop decision, and risk.
- Storyboard pages pair visual sources with copy and frame notes.
- Entry / Hero / Transition frame cards remain visible.
- Critical-frame mockups remain a required visual section, not text-only notes.
- Long webpage screenshots are manually scrollable, not auto-scrolled.
- Production/sync content is compact and appears after the visual board work.

- 首屏是视觉 contact sheet，不是长摘要。
- 素材先于页面计划 prose 出现。
- 每个素材都有角色、裁切决定和风险。
- 分镜页把视觉来源、文案和 frame notes 放在一起。
- Entry / Hero / Transition frame cards 保持可见。
- Critical-frame mockup 是必需视觉章节，不能只写文字说明。
- 长网页截图手动滚动，不自动滚动。
- Production/sync 内容保持小型，放在视觉方向工作之后。

## Use In A New Run / 新项目如何使用

1. Read the user's brief and real assets.
2. Select a Design Router aesthetic system.
3. Ask mandatory multiple-choice questions for sections, assets, and mainline.
4. Build `direction_board.html` using Pattern A from `references/visual-board-patterns.md`.
5. Borrow the structure of the golden reference, but replace all content, colors, and typography according to the selected project and design route.

## Do Not Copy / 不要复制

- Do not copy the rough board's project-specific screenshots.
- Do not copy its exact colors when another Design Router style is selected.
- Do not copy its page count or narrative structure.
- Do not copy its text.

Copy the **board logic** only: visual first, asset wall, storyboard, frame mockups, compact production handoff.

只复制 **board logic**：视觉优先、素材墙、分镜、关键帧 mockup、小型 production handoff。
