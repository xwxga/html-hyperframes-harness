# Motion Intent Contract

Motion intent is director-level guidance. It explains why motion exists and how it should connect reviewed critical frames.

## Required Fields

- Target frame or asset
- Motion purpose
- Start state
- End state
- Timing
- Constraint

## Rules

- Do not replace critical-frame planning with animation notes.
- Asset motion should make evidence more readable.
- Text motion should support rhythm, not decorate weak copy.
- Final HyperFrames HTML should preserve motion intent as metadata when practical.
