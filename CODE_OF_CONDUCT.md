## Standard / 标准

Contributors should keep discussion technical, specific, and useful. Critique artifacts, contracts, and implementation decisions directly; do not attack people.

贡献者应保持讨论技术化、具体、有效。可以直接批评 artifact、契约和实现决策，但不要攻击人。

## Expected Behavior / 期望行为

- Keep issues and reviews focused on reproducible facts. / Issue 和 review 聚焦可复现事实。
- Respect the project scope and V1 boundaries. / 尊重项目范围和 V1 边界。
- Do not post secrets, private data, or private planning state. / 不发布密钥、私人数据或私有计划状态。
- Use bilingual context where it helps public reuse. / 有助于公开复用时提供中英双语上下文。

## Enforcement / 执行

Maintainers may edit, hide, or remove comments that derail technical work, expose secrets, or introduce personal attacks.

维护者可以编辑、隐藏或删除偏离技术工作、暴露密钥或进行人身攻击的评论。
